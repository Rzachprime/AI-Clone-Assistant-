# Weekly Planning Clone Pack

## Purpose

This pack helps beginners use a personal AI clone to run a steady weekly planning rhythm.

Where the Daily Admin Clone helps with the pressure of the day, the Weekly Planning Clone helps with the shape of the week.

Its job is to help the user:

- reset
- sort open loops
- choose realistic priorities
- reduce hidden overload
- carry the right things forward

## Who This Is For

This pack is for people who do not necessarily need more ambition.

They need more clarity.

It is especially useful for people who:

- end the week with too many loose ends
- start Monday already behind
- carry unfinished tasks forward without reevaluating them
- struggle to tell the difference between urgent and important
- want a calmer weekly reset

## Things You Need Before You Start

Before using this pack, it helps to have:

- an AI tool such as ChatGPT or Codex
- one place to write and save prompts
- access to the user’s calendar and open tasks
- a weekly time block for review

The writing tool can be simple.

People can write and save their prompts in:

- a text editor
- a Word document
- a Notes app
- a Google Doc
- an `.md` file

This is helpful because weekly review prompts often become reusable templates.

## How To Use This With ChatGPT or Codex

The easiest way to use this pack is:

## 1. Set aside one weekly review moment

This could be:

- Friday afternoon
- Sunday evening
- Monday morning before the week begins

Consistency matters more than perfection.

## 2. Gather the real materials first

Before prompting, collect:

- next week’s calendar
- open tasks
- unfinished items from the previous week
- pending follow-ups
- known constraints

## 3. Draft or reuse the weekly prompt

Write the prompt in a file first, then paste it into ChatGPT or Codex with the actual weekly material.

## 4. Review the output for realism

Ask:

- Did it overpack the week?
- Did it catch what is still open?
- Did it help distinguish what matters from what is merely loud?
- Did it surface overload or conflicts?

## 5. Adjust the prompt over time

If the output is too broad, too optimistic, or too generic, refine it.

Usually that means:

- adding constraints
- clarifying priorities
- improving the format
- telling it to optimize for realism instead of ambition

## Why Weekly Planning Matters

Many people do not lose control because of one chaotic day.

They lose clarity because small tasks, loose notes, unfinished follow-ups, and shifting priorities accumulate across the week.

A weekly planning clone helps interrupt that pattern.

It creates a repeatable moment to step back, see the whole picture, and make better decisions before the next week begins.

## What This Pack Is For

This pack is useful for:

- weekly review
- weekly planning
- backlog triage
- carry-forward decisions
- identifying what matters most next week
- catching open loops before they get buried
- preparing a calmer start to Monday

How to use this list:

Choose one or two weekly problems that repeat often.

Do not try to turn the weekly planning clone into a whole life-management system on the first pass.

## What This Pack Is Not For

This pack is not designed to:

- make major life decisions for the user
- commit to deadlines without review
- replace human judgment about tradeoffs
- automatically manage other people’s expectations
- handle sensitive planning without oversight

It is a planning support system.

Not a replacement for responsibility.

## The Core Promise

The Weekly Planning Clone should help the user:

- close the previous week cleanly
- enter the next week with more clarity
- stop carrying every open task in their head
- make better choices about what actually fits
- protect attention before the week gets crowded

That is where the value comes from.

Not more plans.

Better plans.

## What a Good Weekly Planning Clone Actually Does

A strong version of this clone should be able to:

- review what was completed
- identify what is still open
- sort unfinished tasks
- group priorities by urgency and importance
- flag overload
- suggest what should be deferred, dropped, or clarified
- prepare a realistic weekly outline

How to use this list:

Use it as the standard for success.

If the clone is trying to solve ten larger problems beyond this, the setup may be too broad.

## What the User Should Gather Before the Weekly Review

The clone becomes much more useful when the user gives it a full enough picture.

Recommended inputs:

- calendar for the next 7 days
- open tasks
- unfinished items from the prior week
- follow-ups still pending
- important deadlines
- personal constraints or limited availability
- anything causing background stress

How to use this list:

The user does not need every category every time.

But the more the AI understands the real shape of the week, the more realistic the plan becomes.

## A Simple Weekly Planning Clone Prompt

This is a practical starter prompt.

It gives someone a strong first structure they can improve over time.

```text
Act as my weekly planning assistant.

Help me review the past week and prepare for the next one.

My preferences:
- Keep the plan realistic, not idealized.
- Prioritize what matters most, not just what is loudest.
- Flag overload or hidden conflicts clearly.
- Do not invent deadlines or commitments.

Return the answer in these sections:
1. What was completed
2. What is still open
3. What matters most next week
4. What can wait, be delegated, or be dropped
5. Risks, conflicts, or missing information
```

How to use this prompt:

Paste the prompt first, then paste the weekly material underneath it.

Ask the AI to work from reality, not aspiration.

## The Weekly Planning Workflow

This pack works best as a repeatable sequence.

That sequence matters because weekly review is easier when the user knows what kind of thinking should happen in what order.

## Step 1: Look Back

The user provides:

- what was planned
- what actually got done
- what stayed open
- what created friction

The clone should help surface:

- completed work
- unfinished work
- patterns of overcommitment
- recurring friction

Why this step matters:

People often skip directly to next week without understanding why the last week became messy.

## Step 2: Clear the Backlog

The user provides:

- open tasks
- carry-forwards
- loose notes
- follow-ups

The clone should sort them into:

- next week
- later
- no longer necessary
- unclear and needs definition

Why this step matters:

Backlog pressure becomes less stressful when unclear items stop living in the same mental pile as important ones.

## Step 3: Set Weekly Priorities

The user provides:

- upcoming deadlines
- appointments
- limited time blocks
- energy constraints

The clone should return:

- top weekly priorities
- likely pressure points
- what cannot realistically fit
- where focus time needs protection

Why this step matters:

This is where realism protects the week from becoming wishful thinking.

## Step 4: Prepare the Week

The user provides:

- any final notes
- personal constraints
- known unknowns

The clone should return:

- a simple weekly outline
- suggested focus areas
- follow-ups to prepare
- items needing clarification early

Why this step matters:

The goal is not to plan every hour.

The goal is to start the week with better shape and less mental clutter.

## Recommended Output Formats

Format matters here because weekly planning can easily become too abstract.

The right structure makes the output easier to use.

## Format 1: Weekly Review

- What got done
- What is still open
- What created friction
- What I learned

Use this when:

The user mainly wants perspective and cleanup.

## Format 2: Weekly Planning

- Top priorities
- Important but flexible items
- Follow-ups
- Risks or overload
- What can wait

Use this when:

The user wants a practical preview of the coming week.

## Format 3: Weekly Triage

- Must happen
- Should happen
- Can wait
- Needs clarification
- Drop or defer

Use this when:

The user is overloaded and needs decisions more than detail.

## Prompt Variations for Common Weekly Planning Tasks

These prompts are best used as templates.

Someone should copy the one closest to their need, test it on a real example, and then improve it based on what happens.

## A. Weekly Reset Prompt

Use this when the user wants to close the previous week clearly.

```text
Act as my weekly reset assistant.

Review the information below and help me close out the week clearly.

Return:
1. What got done
2. What is still open
3. What should carry forward
4. What should be dropped or delayed
5. What seems to be creating repeated friction
```

## B. Weekly Priority Prompt

Use this when the user mainly wants help deciding what matters next week.

```text
Act as my planning assistant.

Based on my upcoming calendar, tasks, and deadlines, help me choose the most important priorities for next week.

Please:
- keep the plan realistic
- point out overload
- separate must-do items from optional items

Return:
1. Core priorities
2. Important but flexible work
3. Conflicts or risks
4. What should wait
```

## C. Backlog Triage Prompt

Use this when the user has too many open tasks and needs categories before planning.

```text
Act as my task triage assistant.

Help me sort this backlog into:
- next week
- later
- delegate
- drop
- unclear

If something is vague, place it under unclear instead of guessing.
```

## D. Weekly Focus Protection Prompt

Use this when the user wants to protect time and reduce crowding.

```text
Act as my planning partner.

Help me protect my attention next week.

Using the calendar and task list below:
- identify where focused work is most needed
- identify where meetings or obligations may crowd it out
- suggest what should be moved, shortened, or deprioritized

Keep the advice practical and grounded.
```

## Common Failure Points

These problems are common and correctable.

They usually come from unclear constraints or unrealistic planning habits, not from a lack of model quality.

## 1. Planning an Ideal Week Instead of a Real Week

Problem:

The user ends the planning session with a plan that looks good but does not fit reality.

Fix:

Tell the clone to optimize for realism, not ambition.

Why this works:

A realistic plan is easier to trust and easier to follow.

## 2. Carrying Too Much Forward

Problem:

The user moves unfinished work forward without reevaluating it.

Fix:

Require the clone to label items as:

- still important
- unclear
- no longer necessary

Why this works:

It forces decisions instead of allowing backlog drift.

## 3. Missing Energy or Capacity Constraints

Problem:

The weekly plan ignores travel, fatigue, family constraints, or meeting load.

Fix:

Add personal constraints explicitly to the prompt.

Why this works:

Time is not the only real limit.

Energy and attention matter too.

## 4. Confusing Urgency With Importance

Problem:

The plan becomes reactive.

Fix:

Ask the clone to distinguish:

- urgent
- important
- loud but low-value

Why this works:

This is one of the most useful distinctions in weekly planning.

## Weekly Review Checklist

Before using a weekly planning prompt, ask:

1. Did I gather the real weekly inputs?
2. Did I give the clone enough constraints?
3. Did I tell it to optimize for realism?
4. Did I ask for a useful format?
5. Am I using this to clarify, not to hand over judgment?

If the answer is mostly yes, the prompt is likely ready.

## Final Principle

A Weekly Planning Clone should not make the week look impressive.

It should make the week easier to see clearly.
